# Plan 02-01: Boost State & Transformations

Implement the core Boost state lifecycle, yellow visual set, and specialized scoring logic.

## Objective
Enable a temporary 10-second "Boost State" that transforms all gameplay into a high-scoring, fixed-duration mode.

<task id="2" type="execute">
<title>Refactor GameManager for Boost State</title>
<read_first>
- Assets/Scripts/GameManager.cs
</read_first>
<action>
1. Add `bool _isBoostActive` and `bool _hasBoostTriggeredThisStage` flags.
2. Update `HandleLanternHit` to award **10 points** for `Boost` type.
3. Update `HandleLanternEscaped` to award **0 points** for `Boost` type.
4. Implement `ActivateBoostMode()`:
    - Set `_isBoostActive = true` and `_hasBoostTriggeredThisStage = true`.
    - Call `spawnerSystem.ClearAllLanterns()` (Pop all with 0 points).
    - Start a 10s coroutine `BoostStateCoroutine()`.
5. Implement `BoostStateCoroutine()`:
    - Trigger `spawnerSystem.StartBoostSpawning()`.
    - Wait 10 seconds.
    - Set `_isBoostActive = false`.
    - Check if `StageScore >= CurrentThreshold()`. If so, `StartIntermission()`.
    - Otherwise, resume normal spawning for the remaining stage time.
6. Update `Update()` Timer Logic:
    - When `TimeRemaining <= 0`:
    - If `CurrentStage == Stage1` or `Stage2`:
        - If `StageScore < CurrentThreshold()`, trigger `GameOver()`.
        - Else, `StartIntermission()`.
    - If `CurrentStage == Stage3`, trigger `GameOver()`.
</action>
<acceptance_criteria>
- `GameManager` awards 10 pts for Boost hits.
- `BoostStateCoroutine` runs for exactly 10s.
- `ActivateBoostMode` clears all current lanterns.
</acceptance_criteria>
</task>
