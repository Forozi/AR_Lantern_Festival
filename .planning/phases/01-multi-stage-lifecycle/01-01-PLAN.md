# Plan 01-01: Multi-Stage Lifecycle Management

Implement a 3-stage game state machine in `GameManager` with support for score-based skipping and timed transitions.

## Objective
Establish the structural framework for transitioning between Stage 1, 2, and 3, including the intermission countdown sequence.

## Tasks

<task id="1" type="execute">
<title>Refactor GameManager for Stage State</title>
<read_first>
- Assets/Scripts/GameManager.cs
</read_first>
<action>
1. Define a `public enum GameStage { Stage1, Stage2, Stage3, Intermission }`.
2. Add a private `GameStage _currentStage` field.
3. Add a `public int StageScore` property to track points gained within the current stage.
4. Update `Start()` to initialize stage to `Stage1` and `StageScore` to 0.
</action>
<acceptance_criteria>
- `GameManager.cs` contains the `GameStage` enum.
- `GameManager.cs` contains a `StageScore` property.
</acceptance_criteria>
</task>

<task id="2" type="execute">
<title>Implement Stage Skip Logic</title>
<read_first>
- Assets/Scripts/GameManager.cs
</read_first>
<action>
1. Create a `CheckStageProgression()` method.
2. If `StageScore >= 50` and current stage is not `Stage3`, call `StartIntermission()`.
3. Update `HandleLanternHit` and `HandleLanternEscaped` to increment/decrement `StageScore` as well as the global score.
4. Ensure `StageScore` resets to 0 when starting a new stage.
</action>
<acceptance_criteria>
- `GameManager.cs` contains `CheckStageProgression` method.
- `HandleLanternHit` updates both global score and `StageScore`.
</acceptance_criteria>
</task>

<task id="3" type="execute">
<title>Implement Intermission & Timer Reset</title>
<read_first>
- Assets/Scripts/GameManager.cs
- Assets/Scripts/SpawnerSystem.cs
</read_first>
<action>
1. Create `StartIntermission()` and `AdvanceStage()` methods.
2. `StartIntermission()` should:
    - Set state to `Intermission`.
    - Call `spawnerSystem.ClearAllLanterns()`.
    - Start a coroutine for a 3-second countdown.
3. `AdvanceStage()` should:
    - Increment stage (Stage 1 -> Stage 2 -> Stage 3).
    - Reset `TimeRemaining` to 30.0f.
    - Reset `StageScore` to 0.
    - Set state to the new stage.
</action>
<acceptance_criteria>
- `GameManager.cs` contains `StartIntermission` and `AdvanceStage` methods.
- Timer resets to 30.0f upon `AdvanceStage`.
</acceptance_criteria>
</task>
