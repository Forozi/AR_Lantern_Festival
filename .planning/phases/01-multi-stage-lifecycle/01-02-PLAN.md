# Plan 01-02: Spawner Difficulty Scaling

Implement speed multipliers and stage-based difficulty ramping in the `SpawnerSystem`.

## Objective
Enable the game to increase lantern speed dynamically based on the current stage provided by the `GameManager`.

## Tasks

<task id="1" type="execute">
<title>Add Speed Multiplier to LanternBehaviour</title>
<read_first>
- Assets/Scripts/LanternBehaviour.cs
</read_first>
<action>
1. Add a `public float currentSpeedMultiplier = 1.0f;` field to `LanternBehaviour`.
2. Update the movement logic (likely in `Update()`) to multiply the base speed by `currentSpeedMultiplier`.
</action>
<acceptance_criteria>
- `LanternBehaviour.cs` contains `currentSpeedMultiplier` field.
- Movement calculation includes the multiplier.
</acceptance_criteria>
</task>

<task id="2" type="execute">
<title>Link Spawner to GameManager Stage</title>
<read_first>
- Assets/Scripts/SpawnerSystem.cs
- Assets/Scripts/GameManager.cs
</read_first>
<action>
1. Add a `GetSpeedMultiplierForStage()` helper in `SpawnerSystem` that returns:
    - Stage 1: 1.0f
    - Stage 2: 1.5f
    - Stage 3: 2.5f
2. Update `SpawnAt(SpawnPoint point)` to set the `currentSpeedMultiplier` on the spawned lantern instance using the value from the current stage.
</action>
<acceptance_criteria>
- `SpawnerSystem.cs` retrieves stage info from `GameManager`.
- Spawned lanterns have speed multipliers set to 1.5f in Stage 2 and 2.5f in Stage 3.
</acceptance_criteria>
</task>

3. Advance to Stage 3 and verify lanterns are very fast (2.5x).
